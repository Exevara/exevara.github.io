Arguments:

Runtime

Overview

The Exevara runtime is a shared daemon that handles package installation, dependency isolation, caching, and execution on behalf of apps built with Gogo. Apps become lightweight clients — the runtime provides the heavy infrastructure once, shared across all Exevara apps on the machine.

Paths

The runtime stores data under the user's home directory:

  • ~/.exevara/ — root runtime directory
  • ~/.exevara/envs/root/ — shared dependency environment
  • ~/.exevara/envs/<uuid>/ — isolated environment for conflicting dependencies

Environments

The runtime manages two types of environments:

  • Root environment — shared across all apps. Used when dependencies are compatible.
  • Isolated environment — created per app when dependency conflicts are detected. Identified by UUID.

Package installation is handled via uv.

Communication

Overview

Gogo communicates with the Exevara runtime over a local socket connection. When your app calls a function via Gogo, the request is serialized and sent to the runtime, executed there, and the result is returned to your app.

Errors

RemoteFunctionError

Raised when a function executed on the Exevara runtime throws an exception. The error includes a traceback from the runtime side to help identify the source of the failure.

This error originates in the developer's code — it indicates a bug in the app, not a problem with the user's machine or the runtime itself.
